# Канонический Пример Смешанной Зрелости

Этот пример специально смешанный. Он показывает:

- структурированный модуль
- явную известную ветку
- грубый модуль для известного, но ещё недоопределённого пути
- грубый модуль внутри основного маршрута

```yaml
modules:
  - id: request-intake
    purpose: Нормализовать входящий запрос и решить, может ли обработка продолжиться автоматически.
    input: Сырой запрос
    output: Нормализованный контекст запроса или немедленное отклонение
    steps:
      - id: read-request
        body_markdown: |
          Прочитать входящий запрос и нормализовать payload в стабильный внутренний контекст.
        links:
          - to: validate-fields
      - id: validate-fields
        body_markdown: |
          Проверить поля, которые уже известны как обязательные для дальнейшей обработки.
        links:
          - to: route-request
            condition: запрос приемлем
          - to: reject-request
            condition: запрос сразу невалиден
      - id: route-request
        body_markdown: |
          Направить приемлемые запросы либо в автоматическое обогащение, либо в ветку ручной проверки.
        links:
          - to: enrich-overview
            condition: запрос можно обработать автоматически
          - to: manual-review-overview
            condition: запрос требует ручной проверки
      - id: reject-request
        body_markdown: |
          Остановить обработку и вернуть пользователю понятную причину отклонения.

  - id: enrichment
    purpose: Собрать дополнительный контекст перед финальной генерацией ответа.
    output: Обогащённый контекст выполнения
    steps:
      - id: enrich-overview
        body_markdown: |
          Собрать tenant context, feature flags, compliance tags и formatting rules перед
          передачей запроса в финальную генерацию ответа.

          Точная внутренняя последовательность ещё не согласована, поэтому этот модуль нужно
          оставить одним грубым логическим блоком вместо выдумывания детальных шагов.
        links:
          - to: finalize-response
            condition: обогащение завершено

  - id: response-finalization
    purpose: Собрать финальный ответ из подготовленного контекста.
    steps:
      - id: finalize-response
        body_markdown: |
          Сформировать финальный ответ из нормализованного и обогащённого контекста.

  - id: manual-review
    purpose: Обработать high-risk запросы, которые должны выйти из автоматического пути.
    steps:
      - id: manual-review-overview
        body_markdown: |
          Известно, что эта ветка существует, но её внутренний workflow ещё не согласован.
          Держи её как единый логический блок, пока путь ручной проверки не будет обсуждён подробнее.
```
