# Ошибки при работе ресёрч-агентов

## 2026-03-20: Nike Manoa Leather v1 — игнорирование инструкций

**Задача:** Найти прямые ссылки на Nike Manoa Leather чёрные 43 EU, проверить наличие на каждом сайте.

**Что пошло не так:**
- Агент НЕ дал прямых ссылок на товары — только общие выводы
- Агент НЕ зашёл на сайты и не проверил реальное наличие 43 размера
- В research.md не записал найденные данные (файл остался пустым)
- По сути проигнорировал ключевую часть задания: "прямая ссылка на страницу товара" и "подтверждённое наличие 43 размера"
- Сослался на "блокировку серверных IP" как причину — но даже те сайты что доступны не были проверены

**Что должен был сделать:**
- Зайти на каждый сайт через web_fetch / browser
- Найти конкретные страницы товаров
- Записать URL, цену, наличие 43
- Если сайт блокирует — записать это и попробовать через поисковые запросы найти прямые ссылки

**Модель:** GPT-5.4, thinking: xhigh
**Время работы:** ~16 минут
**Результат:** неудовлетворительный — пришлось искать вручную

## 2026-03-20: Nike Manoa Leather v2 — повтор той же ошибки

**Задача:** То же самое, явно указано BROWSER_FIRST, проверить каждый сайт.

**Что пошло не так:**
- Снова не записал прямые ссылки в research-v2.md (файл остался с пустыми секциями)
- progress.md показывает что агент нашёл информацию, но не записал её в файл результатов

**Вывод:** Агент-исследователь плохо выполняет задачи требующие конкретных проверок на сайтах. Лучше работает для сбора общей информации через поиск.

## Паттерны ошибок

1. **Не записывает результаты в файл** — находит данные но не пишет в research.md
2. **Игнорирует конкретные инструкции** — просят прямые ссылки, даёт общие выводы
3. **Ссылается на блокировки** — вместо того чтобы попробовать альтернативные способы
4. **Не использует browser** — даже когда явно указано BROWSER_FIRST
