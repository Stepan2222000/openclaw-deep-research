### Субагенты — делегирование работы через Codex

Когда у тебя много работы которую можно распараллелить — делегируй субагентам. Каждый субагент работает автономно и записывает результат в свой файл.

#### Когда использовать

- Много кандидатов/источников для анализа (30+)
- Нужно извлечь данные из множества сайтов параллельно
- Задача легко делится на независимые части

#### Запуск субагента

```bash
codex exec --full-auto --skip-git-repo-check -C /root/results/<slug> -o /root/results/<slug>/agent-<N>.md "ПРОМПТ" &
```

- `--full-auto` — без подтверждений, автономная работа
- `--skip-git-repo-check` — работать без git-репозитория
- `-C <dir>` — рабочая директория субагента
- `-o <file>` — записать финальный ответ в файл
- `&` — запуск в фоне (для параллельной работы)

#### Формат промпта субагенту

Промпт должен быть подробным — субагент не знает контекст исследования. Обязательно включи:

1. **Роль и задача** — что конкретно должен сделать
2. **Контекст** — brief исследования, что мы ищем и зачем
3. **Жёсткие требования** — конкретные, измеримые критерии которые обязательно проверить
4. **Список для обработки** — конкретные кандидаты/URL/источники
5. **Формат результата** — как именно записать результат (что включить, в каком виде)
6. **Инструменты** — указать: `Прочитай файл /root/.openclaw/workspace-researcher/TOOLS.md — там инструкции по всем доступным инструментам (Exa, ScrapFly, Brave и др.). Используй их для получения контента страниц. НЕ используй curl/wget/requests для получения HTML.`

#### Параллельный запуск

```bash
# Подготовка
mkdir -p /root/results/<slug>

# Запуск субагентов параллельно (каждый со своим промптом и группой)
codex exec --full-auto --skip-git-repo-check -C /root/results/<slug> -o /root/results/<slug>/agent-01.md "Промпт для группы 1..." &
codex exec --full-auto --skip-git-repo-check -C /root/results/<slug> -o /root/results/<slug>/agent-02.md "Промпт для группы 2..." &
codex exec --full-auto --skip-git-repo-check -C /root/results/<slug> -o /root/results/<slug>/agent-03.md "Промпт для группы 3..." &

# Дождаться завершения всех
wait
```

#### Сбор результатов

После завершения субагентов:
1. Прочитай все файлы `/root/results/<slug>/agent-*.md`
2. Агрегируй результаты — собери подходящих кандидатов
3. Перепроверь ключевые данные если есть сомнения
4. Запиши итог в research.md

#### Правила

- Давай субагентам **подробный промпт** — они не знают контекст
- Всегда указывай **путь к TOOLS.md** — субагент должен знать как пользоваться инструментами
- Включай **жёсткие требования** в промпт — субагент проверяет каждое
- Группы по **10-15 элементов** — слишком много = поверхностный анализ
- **Не запускай больше 10 субагентов одновременно** — ограничение ресурсов
