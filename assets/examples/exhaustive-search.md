### Пример: исчерпывающий поиск вариантов

**Задача:** Найти все подходящие сервисы managed-хостинга для базы данных в Европе дешевле $25/мес на 100 GB. Минимальное время работы: 2 часа.

---

**Неправильно (что произошло):**

Агент начал работу. Сделал 3 поисковых запроса через Exa: "managed database hosting Europe", "serverless database pricing", "cheap managed PostgreSQL". Из первых результатов выбрал 15 сервисов — самые известные (AWS, Azure, GCP, DigitalOcean, Hetzner, Scaleway, Neon, Aiven, Crunchy Bridge, Heroku, Vultr, Linode, OVH, Supabase, PlanetScale). Проверил pricing у каждого. Записал сравнительную таблицу. Написал FINAL через 35 минут.

Проблемы:
- Сделал всего 3 поисковых запроса — а мог десятки, комбинируя разные формулировки
- Посмотрел только первые 5-10 результатов из каждого запроса — а Exa может вернуть 30-100 за один вызов
- Не искал каталоги, директории, awesome-lists, comparison pages
- Не пробовал региональные запросы (на немецком, французском, по конкретным странам)
- Не искал community-обсуждения (Reddit, HN, форумы)
- Остановился через 35 минут при выделенных 2 часах — решил что "основные игроки найдены"
- 15 сервисов — это то, что он и так знал. Реального поиска почти не было.

**Что обнаружилось потом:**

Когда заставили продолжить поиск — выяснилось:
- Один запрос "database hosting providers directory" привёл на официальную директорию с 200+ провайдерами
- Awesome-list на GitHub содержал ещё 50 малоизвестных сервисов
- Поиск "managed database France / Germany / Netherlands / Finland" дал локальных провайдеров которых нет в англоязычной выдаче
- Comparison-сайты (G2, DB-Engines, AlternativeTo) дали ещё десятки
- Reddit-треды "what do you use for database hosting" упоминали нишевые сервисы
- Exa с `numResults: 50` по разным запросам выдавал уникальные результаты, которых не было в первых 10

Итого вместо 15 было найдено 350+ провайдеров. Из них 30 подходили по условиям — и среди них были варианты значительно дешевле тех 15, с которых агент остановился.

**Агент пропустил 95% рынка потому что:**
1. Искал слишком мало — 3 запроса вместо 30-50
2. Смотрел слишком мало результатов — первые 10 вместо 30-100 на запрос
3. Не комбинировал запросы — не менял формулировки, не добавлял ключевые слова, не искал на других языках
4. Не искал каталоги — один каталог стоит десяти обычных поисков
5. Остановился рано — потратил 35 минут из 2 часов и решил что "всё нашёл"

---

**Правильно:**

1. **Начать с широкого поиска, постоянно меняя запросы.** Не 3 запроса, а 20-30+. Каждый раз другая формулировка:
   - "managed database hosting Europe"
   - "database as a service providers list"
   - "DBaaS comparison 2026"
   - "cheap managed PostgreSQL Europe"
   - "managed database providers directory"
   - "alternatives to [каждый найденный сервис]"
   - "database hosting [конкретная страна]"
   - Использовать `numResults: 30-50` в каждом запросе, а не 5-10

2. **Искать каталоги и агрегаторы — это главный источник.** Один каталог = десятки-сотни кандидатов:
   - Официальные директории технологии ("certified hosting providers", "cloud partners")
   - Awesome-lists на GitHub
   - Comparison-сайты: G2, AlternativeTo, Capterra, DB-Engines
   - Региональные каталоги облачных провайдеров

3. **Работать всё выделенное время.** Если дали 2 часа — искать 2 часа. Волнами:
   - Первые 30 мин: широкий поиск, сбор кандидатов
   - 30-60 мин: каталоги, директории, awesome-lists
   - 60-90 мин: региональный поиск, community, нишевые запросы
   - 90-120 мин: углублённый анализ подходящих, проверка жёстких требований
   - После каждой волны: "что я ещё не проверял?" → новая волна

4. **Собрать полный список ПЕРЕД анализом.** Не анализировать первые 15 и останавливаться — сначала найти всех. Список из 200-300 кандидатов — нормально.

5. **Если кандидатов много (30+) — делегировать субагентам.** Разбить список на группы, каждому субагенту дать группу для проверки по жёстким требованиям.

---

**Ключевой принцип:** 15 популярных сервисов — это не "все", это 5% рынка. Настоящий исчерпывающий поиск — это десятки запросов, каталоги и директории, региональный поиск, community-обсуждения. Если выделено 2 часа — работай 2 часа. За 35 минут невозможно найти всё. Каждая новая волна поиска приносит новых кандидатов, которых не было в предыдущих. Не останавливайся.
